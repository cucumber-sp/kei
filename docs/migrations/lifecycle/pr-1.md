## PR scope

**Implements:** `docs/design/lifecycle-module.md#7` (PR 1).

This PR moves the **Decide** sub-concern of lifecycle hook
auto-generation out of `src/checker/struct-checker.ts` (pass 1.5)
into a new `src/lifecycle/` module. Behaviour-preserving ‚Äî every
struct that previously got an auto `__destroy` / `__oncopy` still
does, with the same field set in the same order.

**Files affected:**

- **NEW** `compiler/src/lifecycle/index.ts` ‚Äî public interface:
  `createLifecycle()` factory; `register(struct)`, `runFixedPoint()`,
  `hasDestroy(struct)`, `hasOncopy(struct)`, `getDecision(struct)`
  methods.
- **NEW** `compiler/src/lifecycle/types.ts` ‚Äî the
  `LifecycleDecision` and `ManagedFieldRef` types per design doc ¬ß4.
- **NEW** `compiler/src/lifecycle/decide.ts` ‚Äî the fixed-point
  iteration; moved verbatim from `struct-checker.ts`'s pass-1.5
  helpers, then refactored to read/write through the module's
  internal `Map<StructType, LifecycleDecision>` instead of mutating
  `structType.methods`.
- **MODIFIED** `compiler/src/checker/struct-checker.ts` ‚Äî the
  `autoGenerateDestroys` / `autoGenerateOncopies` methods now
  delegate to `lifecycle.runFixedPoint()`. They still register
  the auto-generated method types on `structType.methods` (so
  type-checking call sites that reference `s.__destroy()` still
  works) ‚Äî that mutation is **temporarily kept** as a transition
  shim. PR 4 will remove it once Diagnostics-style queries replace
  the type-table lookups.
- **MODIFIED** `compiler/src/checker/checker.ts` ‚Äî constructs a
  `Lifecycle` instance and threads it into `struct-checker`'s
  context. Pattern: same as how `Diagnostics`'s `Collector` will
  thread later.

**Out of scope (do not touch in this PR):**

- The Synthesise sub-concern (the KIR function-body generator)
  stays in `kir/lowering-struct.ts` ‚Äî moves in PR 2.
- The Insert sub-concern stays scattered as today ‚Äî markers and
  the rewriting pass are PR 3+.
- Don't introduce marker IR instructions yet.
- Don't touch `kir/lowering-*.ts` or `backend/`.
- Don't remove the `structType.methods.set("__destroy", ‚Ä¶)` calls
  yet ‚Äî they're load-bearing for type-checking until PR 4.
